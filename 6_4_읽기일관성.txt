사용자는 다음과 같은 두가지 방식으로 데이터베이스를 사용한다
1) 읽기 (SELECT)
2) 쓰기 (INSERT, UPDATE, DELETE)

읽기 일관성이란 
쓰기작업을 시작하기 전 마지막 커밋 시점의 상태대로 
다른사람들이 데이터를 볼 수 있도록 보장하는 것을 말한다
  읽는 사람은 쓰는 사람의 작업이 완료되길 기다릴 필요가 없다
  쓰는 사람은 읽는 사람의 작업이 완료되기를 기다릴 필요가 없다
  쓰는 사람은 다른 쓰는 사람의 작업이 완료되기를 기다려야 한다

읽기 일관성 시나리오(시간흐름) 
1.사용자A 
     쓰기 작업을 수행하면 데이터가 변경되기 전에 데이터의 복사본을 만들어 이를 "언두 세그멘트"(오라클)/ "언두로그"(innoDB엔진)에 기록한다
     쓰기 작업으로 실제 데이터 블록이 변경되었다
2. 사용자 B
***쓰기 작업을 수행한 사람외의 모든 읽는 사람에게는 "언두 세그멘트"의 데이터 스냅샷이 표시된다***
3-1. 사용자 A 
     커밋하면 변경된 내용이 영구적으로 저장된다
     언두로그 삭제된다

3-2. 사용자 A 
     롤백하면 언두로그에 들어 있던 이전 데이터가 테이블에 다시 기록된다
     언두로그 삭제된다 
4. 사용자 B
쓰기 작업을 수행한 사람외의 모든 읽는 사람에게는 "언두 로그"가 삭제되었으니, 실제 데이터 블록에서 읽어온다






읽기 일관성은 동일한 데이터에 대해 다음 사항을 보장한다
스냅샷은 특정 시점의 데이터베이스의 읽기 전용 복사본
1) 데이터베이스 백업
2) 트랜잭션 격리 수준
트랜잭션 격리 수준 중 하나인 Snapshot Isolation에서 스냅샷은 트랜잭션 간의 읽기/쓰기 충돌을 방지하고, 데이터베이스의 일관성을 유지
3) 읽기 전용 복사본
 트랜잭션 중에 데이터베이스의 상태를 스냅샷으로 저장한 후, 트랜잭션이 완료될 때까지 다른 작업에서 이 스냅샷을 참조
